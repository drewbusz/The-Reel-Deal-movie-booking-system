This folder contains the final documentation materials for The Reel Deal movie booking system.

Included Files:

System_requirements_documentation.pdf
This document contains the final system requirements documentation for the project. It includes the customer problem statement, glossary, functional and nonfunctional requirements, use cases, UML diagrams, system sequence diagrams, activity diagrams, user interface specification, traceability matrix, system architecture and design, algorithms and data structures, user interface design and implementation, design of tests, project plan, and references.

Brochure_flyer.pdf
This document is the final product brochure for The Reel Deal. It provides a short marketing-style overview of the system, including the project description, core features, system requirements, customer booking process, staff features, business value, project webpage, and demo video link.

Presentation_slides.pdf
This document contains the final presentation slides for the project demo. The slides summarize the product overview, demo roadmap, customer workflow, seat reservation process, staff tools, database design, and final system value.

Purpose of Documentation:
The documentation explains both the design and final implementation of The Reel Deal. It shows how the project moved from system requirements and planning into a working Flask and MySQL application with customer booking, staff ticket validation, staff booking records, and movie/showtime management.

Final Notes:
These documents are intended to support the final demo submission. The source code, testing materials, and data setup files are included in the other project folders.
